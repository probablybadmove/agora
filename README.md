# Agora 🏛️

**The public square for Claude Code plugins.** Browse skills and subagents, then install them in one command.

Agora is two things at once:

1. **A website** — a fast, browsable catalog (web + iOS + Android, one codebase) hosted at **[agora.gagansingh.tech](https://agora.gagansingh.tech)**.
2. **A real Claude Code marketplace** — add it with `/plugin marketplace add probablybadmove/agora` and install any of its plugins.

The catalog UI and the installable marketplace are backed by **the same plugin data**, so they can never drift.

## Tech stack

- **Expo SDK 56 + Expo Router** — file-based routing, universal (web + native) from one codebase
- **React Native Web** — the website is the same app, statically pre-rendered per route for SEO
- **TypeScript** throughout
- **Supabase (Postgres)** — optional live catalog backend (the site ships with a bundled catalog and works without it)
- **Vercel** — static hosting + custom domain
- **EAS** — native iOS/Android builds

## Architecture

```
src/data/catalog.json          ← single source of truth (plugins + categories)
        │
        ├─► src/data/seed.ts            typed view → the website renders from this instantly
        ├─► .claude-plugin/marketplace.json   relative sources  (git-based `/plugin marketplace add`)
        ├─► public/marketplace.json           git-subdir sources (hosted-URL `/plugin marketplace add`)
        └─► supabase/seed.sql                 INSERTs for the optional database

         all four are regenerated by:  npm run build:catalog
```

The website reads the bundled catalog by default. If `EXPO_PUBLIC_SUPABASE_URL` / `EXPO_PUBLIC_SUPABASE_ANON_KEY` are set, it serves the catalog live from Supabase instead, falling back to the bundle on any error.

### Project layout

```
src/app/                 Expo Router screens (/, /browse, /docs, /plugin/[slug], /category/[slug])
src/components/          UI: header, footer, cards, code blocks, markdown, brand mark…
src/data/                catalog.json (source of truth), seed.ts, copy.ts, types.ts
src/lib/supabase.ts      optional Supabase client (publishable/anon key only)
plugins/                 the actual Claude Code plugins (each installable)
.claude-plugin/          marketplace.json (repo-root, relative sources)
public/                  marketplace.json (hosted, git-subdir sources), favicon, robots
supabase/                migrations/0001_init.sql + generated seed.sql
scripts/build-catalog.mjs   regenerates all derived artifacts from catalog.json
```

## Local development

```bash
bun install              # or npm install
npm run web              # dev server (web)
npm run ios              # iOS simulator
npm run android          # Android emulator
```

Edit the catalog in `src/data/catalog.json`, then:

```bash
npm run build:catalog    # regenerate marketplace.json (x2) + supabase/seed.sql
```

## Using the marketplace in Claude Code

```bash
# add the marketplace (git form — recommended)
/plugin marketplace add probablybadmove/agora

# …or from the hosted manifest
/plugin marketplace add https://agora.gagansingh.tech/marketplace.json

# install a plugin, then reload
/plugin install conventional-commits@agora
/reload-plugins
```

## Publish your own plugin

1. Fork the repo and add `plugins/<your-plugin>/` (a `.claude-plugin/plugin.json` plus `skills/` or `agents/`).
2. Add an entry to `src/data/catalog.json` and run `npm run build:catalog`.
3. Open a PR. See `/docs` on the site for the full walkthrough.

## Deploy

### Website → Vercel + agora.gagansingh.tech

`vercel.json` is preconfigured (build command, `dist` output, clean URLs, dynamic-route rewrites, JSON headers for the hosted marketplace manifest).

```bash
npm i -g vercel
vercel            # link the project (first run)
vercel --prod     # deploy
```

Then add `agora.gagansingh.tech` as a domain in the Vercel project and point your DNS at Vercel (an `A`/`CNAME` record per Vercel's instructions). Once live, both the site and `https://agora.gagansingh.tech/marketplace.json` are served.

> If you fork, update the repo slug in `src/data/catalog.json` (`marketplace.repo` / `repoUrl`) and re-run `npm run build:catalog` so the git-subdir sources point at your repo.

### Optional: live catalog → Supabase

```bash
# apply schema + seed to your Supabase project
supabase link --project-ref <ref>
supabase db push                       # runs supabase/migrations/0001_init.sql
psql "$DATABASE_URL" -f supabase/seed.sql   # or run it in the SQL editor
```

Then set the two `EXPO_PUBLIC_*` vars (see `.env.example`) in Vercel and redeploy. Only the **publishable/anon** key is used client-side; the catalog tables are public read-only via RLS.

### Native apps → EAS

```bash
npm i -g eas-cli
eas build --platform ios       # or android
eas submit                     # to the App Store / Play Store
```
